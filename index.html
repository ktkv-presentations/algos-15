Классы объектов

Класс

шаблон или чертеж для создания объектов, которые представляют сущности реального мира

создаётся с помощью ключевого слова class 


class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model

Зачем нужны классы

Разделение ответственности

Каждый класс может отвечать за конкретный функционал, что облегчает поддержку и модификацию кода

Упрощение отладки

Логика, сосредоточенная в классах, позволяет легче идентифицировать и исправлять ошибки

Снижение дублирования кода

Использование наследования и полиморфизма позволяет избежать дублирования кода и делает его более читаемым

Объект

конкретный экземпляр класса, который содержит специфические значения для атрибутов, определённых в классе 


class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model
        
my_car = Car("red", "Ford Mustang")

my_car является объектом класса Car

Атрибуты класса

переменные, которые хранят данные о состоянии объекта

Они могут быть определены как атрибуты экземпляра или атрибуты класса

Атрибуты экземпляра

атрибуты уникальны для каждого объекта и определяются в методе __init__() 


class Car:
    def __init__(self, color, model):
        self.color = color  # Атрибут экземпляра
        self.model = model  # Атрибут экземпляра
        
my_car = Car("red", "Ford Mustang")
another_car = Car("blue", "Chevrolet Camaro")

color и model - атрибуты экземпляра

Атрибуты класса

общие для всех экземпляров класса и могут быть определены вне методов 


class Car:
    wheels = 4  # Атрибут класса

    def __init__(self, color, model):
        self.color = color
        self.model = model
        
my_car = Car("red", "Ford Mustang")
another_car = Car("blue", "Chevrolet Camaro")

print(my_car.wheels)  # Вывод: 4
print(another_car.wheels)  # Вывод: 4

wheels - атрибут класса

Методы

функции, которые определены внутри класса и могут работать с атрибутами объекта или класса

Методы атрибута

методы, которые могут обращаться к атрибутам экземпляра и изменять их 


class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model
        
    def display_info(self):
        print(f"Color: {self.color}, Model: {self.model}")
        
my_car = Car("red", "Ford Mustang")
my_car.display_info()  # Вывод: Color: red, Model: Ford Mustang

Методы класса

методы, которые работают с атрибутами класса и могут изменять состояние класса 


class Car:
    wheels = 4  # Атрибут класса

    @classmethod
    def get_wheels(cls):
        return cls.wheels
        
print(Car.get_wheels())  # Вывод: 4

определяются с помощью декоратора @classmethod

Статические методы

не зависят от состояния объекта или класса


class Car:
    @staticmethod
    def is_motorized():
        return True
        
print(Car.is_motorized())  # Вывод: True

определяются с помощью декоратора @staticmethod

Наследование классов

механизм, позволяющий создавать новый класс на основе существующего, наследуя его атрибуты и методы 


class Vehicle:  # Родительский класс
    def __init__(self, color):
        self.color = color
        
    def start(self):
        print("Vehicle is starting.")

class Car(Vehicle):  # Порожденный класс
    def __init__(self, color, model):
        super().__init__(color)  # Вызов конструктора родительского класса
        self.model = model

    def start(self):
        super().start()  # Вызов метода родительского класса
        print(f"The {self.color} {self.model} car is starting.")

my_car = Car("red", "Ford Mustang")
my_car.start()  
# Вывод:
# Vehicle is starting.
# The red Ford Mustang car is starting.

класс, который наследует другой класс, называется потомком (порожденным классом) , а класс, от которого происходит наследование, называется родительским классом

Переопределение методов

возможность в дочернем классе предоставлять свою реализацию методов, которые уже определены в родительском классе 


class Vehicle:  
    def start(self):
        print("Vehicle is starting.")

class Car(Vehicle):  
    def start(self):  # Переопределение метода
        print("Car is starting.")

my_car = Car()
my_car.start()  
# Вывод: Car is starting.

Множественное наследование

возможность создавать класс на основе более чем одного родительского класса 


class Engine:
    def start_engine(self):
        print("Engine started.")

class Radio:
    def turn_on_radio(self):
        print("Radio turned on.")

class Car(Engine, Radio):  # Множественное наследование
    def drive(self):
        print("Car is driving.")

my_car = Car()
my_car.start_engine()  # Вывод: Engine started.
my_car.turn_on_radio()  # Вывод: Radio turned on.
my_car.drive()          # Вывод: Car is driving.

Спецификаторы доступа

позволяют контролировать, как и где можно обращаться к атрибутам и методам класса

Публичные

Не имеют никаких специальных символов и доступны из любого места

Приватные

не могут быть доступны напрямую за пределами класса. Для обозначения используется два символа _

self.__engine = "V8"

Пример спецификаторов


class Car:
    def __init__(self):
        self.__engine = "V8"  # Приватный атрибут

    def __start_engine(self):  # Приватный метод
        print("Engine started.")

    def start(self):
        self.__start_engine()  # Доступ к приватному методу внутри класса

my_car = Car()
my_car.start()  
# Вывод: Engine started.

конструктор __init__

специальный метод, который автоматически вызывается при создании нового экземпляра класса 


class ClassName:
    def __init__(self, parameters):
        # Инициализация атрибутов объекта

Определение конструктора

Метод __init__ определяется внутри класса 


class ClassName:
    def __init__(self, parameters):
        # Инициализация атрибутов объекта

self : первый параметр метода __init__ всегда должен быть self , который ссылается на создаваемый объект

parameters : дополнительные параметры, которые могут быть переданы в конструктор, позволяющие задавать начальные значения атрибутов

Пример использования конструктора __init__ 


class Person:
    def __init__(self, name, age):
        self.name = name  # Инициализация атрибута name
        self.age = age    # Инициализация атрибута age

    def greet(self):
        print(f"Привет, меня зовут {self.name} и мне {self.age} лет.")

# Создание экземпляров класса Person
person1 = Person("Alice", 30)
person2 = Person("Bob", 25)

# Вызов метода greet
person1.greet()  # Вывод: Привет, меня зовут Alice и мне 30 лет.
person2.greet()  # Вывод: Привет, меня зовут Bob и мне 25 лет.

Особенности конструктора __init__

Необязательные параметры

Конструктор может принимать необязательные параметры с заданием значений по умолчанию 


class Person:
    def __init__(self, name, age=0):  # age по умолчанию 0
        self.name = name
        self.age = age

person3 = Person("Charlie")  # age будет 0
print(person3.age)  # Вывод: 0

Функция super()

встроенная функция в Python, которая используется для вызова методов родительского (или базового) класса из дочернего (или производного) класса


class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)  # Вызов конструктора родительского класса
                        

class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)  # Вызов конструктора родительского класса
        self.breed = breed

    def speak(self):
        return "Woof!"

# Создание экземпляра класса Dog
dog = Dog("Buddy", "Golden Retriever")

# Вызов методов
print(dog.name)   # Вывод: Buddy
print(dog.breed)  # Вывод: Golden Retriever
print(dog.speak())  # Вывод: Woof!
